## 一、数据概况

| 题型 | 题数 | 答案格式 | 选项特征 |
|------|------|----------|----------|
| 单选题 | 89 题 | 单字母（A/B/C/D） | 4 个选项 |

## 二、题型分层展示逻辑

### 2.1 单选题（Single Choice）

**展示要素：**
- 题号标签：第 N 题（N 为当前做题顺序，非原题号）
- 题干文本：去除前缀编号后的纯题干
- 选项列表：A/B/C/D 四个单选按钮（Radio Button 样式）
- 配图：无（本题库无图片题）

**交互规则：**
- 用户点击某个选项后，该选项立即高亮选中（蓝色边框 + 浅蓝背景）
- 再次点击同一选项可取消选择
- 点击其他选项则切换选择
- 选择后按钮不可再更改（锁定状态）

### 2.2 多选题（Multiple Choice）

**展示要素：**
- 题号标签：第 N 题 + 题型徽章 多选题（紫色标签）
- 题干文本
- 选项列表：A/B/C/D 四个复选框（Checkbox 样式）
- 提示信息：题干下方显示灰色小字 （至少选择一项）

**交互规则：**
- 用户可勾选多个选项，每个选项独立切换
- 勾选后选项立即高亮（紫色边框 + 浅紫背景）
- 未提交前可随时增减选项
- 提供"确认提交"按钮（非直接提交）
- 提交后锁定所有选项，不可再修改

### 2.3 判断题（True/False）

**展示要素：**
- 题号标签：第 N 题 + 题型徽章 判断题（橙色标签）
- 题干文本
- 选项列表：仅两个选项 — 正确 / 错误（大号按钮样式）
- 图标辅助：正确按钮带 ✓ 图标，错误按钮带 ✗ 图标

**交互规则：**
- 用户点击 正确 或 错误 按钮之一
- 点击后立即锁定，不可更改
- 按钮样式：选中态高亮，未选态灰色轮廓

## 三、答题交互逻辑（通用流程）

### 3.1 答题流程

`
显示题目 → 用户作答 → 即时校验 → 反馈结果 → 进入下一题
`

### 3.2 即时反馈机制

| 题型 | 反馈时机 | 反馈内容 |
|------|----------|----------|
| 单选题 | 选择后立即 | 绿色边框 + ✓ 标记（正确）或红色边框 + ✗ 标记（错误），同时高亮正确答案 |
| 多选题 | 点击"确认提交"后 | 逐项标记：正确选项绿底、错误选择红底，底部显示完整答案 |
| 判断题 | 选择后立即 | 绿色 ✓ / 红色 ✗ 标记，同时显示标准答案 |

### 3.3 导航控制

- **上一题**：始终可用（除非第一题），可查看已作答题目的历史状态
- **下一题**：
  - 单选题/判断题：作答后即可点击
  - 多选题：提交后即可点击
- **跳过**：提供"跳过此题"按钮，跳过后不计入已答题，可稍后返回
- **题号导航**：底部显示所有题号缩略图，点击可跳转任意题目

## 四、答案校验逻辑

### 4.1 单选题校验

`
用户选项首字母 == 标准答案 → 正确
否则 → 错误
`

**示例：**
- 标准答案：A
- 用户选择：A、Binder → 取首字母 A → 匹配 → 正确
- 用户选择：B、CircuitBreaker → 取首字母 B → 不匹配 → 错误

### 4.2 多选题校验

`
用户选择集合 == 标准答案集合 → 全对（满分）
用户选择集合 ⊂ 标准答案集合 → 部分正确（记半分）
否则 → 错误
`

**计分规则：**
- 完全正确：得 1 分
- 部分正确（选了正确选项但未全选，且未选错误选项）：得 0.5 分
- 选了错误选项：得 0 分

**示例：**
- 标准答案：ABC
- 用户选择 ABC → 全对 → 1 分
- 用户选择 AB → 部分正确 → 0.5 分
- 用户选择 ABCD → 含错误选项 → 0 分
- 用户选择 ACD → 含错误选项 → 0 分

### 4.3 判断题校验

`
用户选择 == 标准答案 → 正确
否则 → 错误
`

**示例：**
- 标准答案：正确
- 用户选择 正确 → 匹配 → 正确
- 用户选择 错误 → 不匹配 → 错误

### 4.4 答案标准化处理

校验前统一处理：
- 单选题：取选项首字母，转大写
- 多选题：提取所有选中选项的首字母，排序后拼接（如 BAC → ABC）
- 判断题：去除空白字符，统一转中文

## 六、错题留存逻辑

### 6.1 错题本机制

**存储方式：** 浏览器 localStorage（无需后端）

**存储结构：**
`json
{
  "questionId": "ms_single_001",
  "type": "single",
  "text": "题干内容",
  "options": ["A、xxx", "B、xxx", ...],
  "correctAnswer": "A",
  "userAnswer": "B",
  "isCorrect": false,
  "wrongCount": 3,
  "lastWrongTime": "2026-07-06T10:30:00+08:00",
  "firstWrongTime": "2026-06-01T08:15:00+08:00"
}
`

**关键逻辑：**
- 每次答错时，若错题本中已有该题 → wrongCount +1，更新 lastWrongTime
- 若错题本中无该题 → 新建条目，wrongCount = 1，记录 irstWrongTime
- 答对时不删除错题记录（保留历史记录），但可标记 mastered 状态

### 6.2 错题练习模式

**触发方式：** 首页"错题重练"入口

**练习规则：**
- 仅从错题本中抽取题目
- 混合所有题型（单/多/判断）
- 可设置抽取数量（全部 / 随机 N 题）
- 答对的题目可标记"已掌握"，暂时移出练习池
- 已掌握题目在 7 天后自动重新加入练习池（间隔重复）

### 6.3 错题统计

- 错题总数
- 高频错题（wrongCount ≥ 2 的题目）
- 各题型错题分布
- 最近一次做错时间

## 七、模式设计

### 7.1 刷题模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 顺序刷题 | 按题库顺序逐题作答 | 初次系统复习 |
| 随机练习 | 打乱顺序，随机抽题 | 巩固记忆 |
| 专项练习 | 仅练习某一题型 | 针对薄弱环节 |
| 错题重练 | 仅练习错题本中的题目 | 考前冲刺 |

### 7.2 练习配置

- **题目数量**：滑块选择（10/20/30/全部）
- **题型筛选**：勾选框（单选/多选/判断）
- **显示图片**：开关（本题库无图片，预留接口）
- **即时反馈**：开关（关闭后所有题做完才显示答案）

## 九、技术实现要点

### 9.1 数据合并

- 英语题库（32 题）+ 微服务题库（214 题）= 246 题
- 统一数据结构，增加 source 字段区分来源
- 多选题和判断题的答案字段统一为字符串类型

### 9.2 状态管理

- 使用 JavaScript 对象管理全局状态
- 答题进度、答案、得分实时同步
- 错题本数据持久化到 localStorage

### 9.3 兼容性

- 纯前端实现，无需后端服务
- 直接双击 index.html 即可运行
- 支持 Chrome / Edge / Firefox 等现代浏览器
